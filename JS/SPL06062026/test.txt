1. var, let, and const

These are used to declare variables.

var
Function scoped
Can be redeclared
Can be reassigned

var a = 10;
var a = 20; // Allowed
let
Block scoped
Cannot be redeclared in the same scope
Can be reassigned
let age = 20;
age = 25; // Allowed
const
Block scoped
Cannot be redeclared
Cannot be reassigned
const pi = 3.14;

2. Hoisting

Hoisting is JavaScript's behavior of allocating memory for variables and functions before code execution starts.

Example:

console.log(a);
var a = 10;

3. Temporal Dead Zone (TDZ)

The TDZ is the period between entering a scope and initializing a let or const variable.

console.log(name);
let name = "Ramya";

5. Scope

Scope determines where a variable can be accessed.

Global Scope
let company = "Google";

function show() {
    console.log(company);
}

Accessible from anywhere.

Function Scope
function test() {
    let age = 23;
}

age is accessible only inside test().

Block Scope
{
    let city = "Chennai";
}

city exists only inside the block.

6. Lexical Scope and Scope Chain

Lexical Scope means a function can access variables from its parent scope.

let a = 10;

function outer() {
    let b = 20;

    function inner() {
        console.log(a);
        console.log(b);
    }

    inner();
}

Output:

10
20
Scope Chain

7. Closure

A Closure is created when an inner function remembers variables from its outer function even after the outer function finishes execution.

function outer() {
    let count = 0;

    return function() {
        count++;
        console.log(count);
    };
}

const counter = outer();

counter();
counter();

Output:

1
2

The inner function remembers count.


8. Why does Closure preserve variables?

Normally local variables are removed after a function completes.

But when an inner function still uses them:

function outer() {
    let balance = 1000;

    return function() {
        console.log(balance);
    };
}

JavaScript keeps the outer function's lexical environment in memory because the returned function still needs it.

This preserved memory is called a Closure.

9. Callback Function

A callback is a function passed as an argument to another function.

function greet() {
    console.log("Hello");
}

function execute(cb) {
    cb();
}

execute(greet);
Why callbacks are used
Asynchronous operations
Event handling
Reusable code

Example:

setTimeout(() => {
    console.log("Data Loaded");
}, 2000);

The arrow function is the callback.

10. Callback Injection

Callback Injection means injecting different behaviors into a function using callbacks.

Example:

function payment(callback) {
    console.log("Payment Successful");
    callback();
}

Different business actions:

function sendEmail() {
    console.log("Email Sent");
}

function generateInvoice() {
    console.log("Invoice Generated");
}

payment(sendEmail);
payment(generateInvoice);

The payment function stays the same, but the behavior changes depending on the callback provided.

This is called Callback Injection.
